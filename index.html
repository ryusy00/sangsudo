시민이 자주 이용하는 기능을 중심으로 설계한 **포털형 메인**입니다. KPI(수질 지표) + 자주찾는 서비스(8개) + 홍보/배너 캐러셀 + 공지 탭을 한 화면에서 확인할 수 있습니다.

  • 바로가기 8개(2×4) 아이콘형 타일 — 서비스/민원으로 즉시 이동
  • KPI 4종(오늘의 평균 pH/잔류염소/탁도/생산량) — 즉시 인지 가능
  • 상수도 소식 배너 캐러셀 — 주요 홍보물/공지 집중 노출
  • 공지/수의계약/입찰/채용 탭 — 최신 글을 한 영역에서 확인
첫 방문자/고객 중심 서비스 바로가기 최적화 브랜드 블루 톤

    ** 디자인 의도 : 디자인 상 공간 절약을 위해 단순 숫자만 배치했습니다.
    ✅"대표전화" 문구 추가 진행
    1.2 바로가기 8개(2×4) 아이콘형 타일이라고 표현되어 있는데 한줄로 표현되어 있는 부분
    ** 디자인 의도 : 기획 의도는 타일(2×4) 배열이었으나 반응형 테스트 과정에서 한 줄 배치된 것 같습니다.
    ✅ 일렬배치에서 2행 4열 형태로 변경
    1.3 공지/수의계약/입찰/채용 <--탭 문구수정 ✅
    1.4 대메뉴가 왼쪽정렬되어 있는 사유가 궁금함.
    ** 디자인 의도 : 메뉴 가독성과 로고/검색 영역과의 균형을 위해 좌측 정렬했습니다.
    ✅ 중앙 정렬· 우측 정렬 모두 가능하므로 피드백에 맞춰 조정 가능합니다.
    1.5 서비스중심의 시안인데 어느 부분을 강조해서 보여주는건지 궁금함.
    ** 디자인 의도 : 자주 찾는 서비스(바로가기 아이콘)와 공지 영역을 가장 먼저 배치해 접근성을 강조했습니다.
    특정 서비스 강조를 원하시면 비주얼(아이콘, 배경 등) 색상 대비로 보완 가능합니다.

좌측 고정 사이드바와 카드 기반의 대시보드형 레이아웃입니다. 실시간 알림띠(긴급/점검), 오늘의 상수도 현황, 정수장 실시간 지표, 상수도 소식 배너, 4분할 공지(공지/수의계약/입찰/채용)를 **한 화면에서 모니터링**할 수 있도록 구성했습니다.

시안 3 · 포털형 시안 3 바로보기 →

시안 3은 포털형(서비스·홍보 허브) 구성을 채택해, 상단의 메인 비주얼/배너와 자주 찾는 서비스 카드, 상수도 소식 및 알림/공고 등 핵심 모듈을 스크롤로 한 페이지에서 연속 탐색할 수 있도록 설계했습니다.

#2563EB (Primary)
#0F172A (Text)
#F6F9FF (Background)
#0b2339 (Footer)
  • 메인 비주얼/배너 캐러셀 — 정책·캠페인·중요 공지의 시선집중과 경로 유도
  • 자주 찾는 서비스 — 카드형 바로가기로 민원 흐름을 최소 클릭으로 연결
  • 상수도 소식 — 공지·홍보 콘텐츠를 모듈화해 빠른 업데이트/순환 노출
  • 알림/공고 카드 4분할(공지/수의계약/입찰/채용) — 각 카테고리를 개별 박스로 구분해 가독성 강화
  • 접근성·반응형 — 키보드 포커스/명도 대비/모바일 재배치 고려
시민 UX 중심 포털형 허브 배너·카드 강조 빠른 경로 유도

    ※ 1차 피드백 피드백 반영전 시안 3 →

    3.1 '한 화면에서 빠르게 탐색' 이라고 써져있는데 알람과소식 공고 부터 내용 짤림.
    --- 디자인 의도 : '한 화면에서 빠르게 탐색'은 스크롤 전 영역만을 뜻한 것이 아니라
    한 페이지 내에서 스크롤로 연속 탐색하는 구조를 의미한 표현이었습니다.
    ✅ 특징 문구를 수정하거나 스크롤 전 영역의 한 화면에 보여지도록 보완하겠습니다.
    3.2 8개의 자주찾는서비스가 3줄로 표현되어 정형화된 느낌이 안듬.
    --- 디자인 의도 : '자주찾는서비스' 우측의 시민참여 홍보를 위해 '상수도 소식' 영역과 동시 배치
    ✅ PC 화면에서는 2행 4열 고정 타일 구조로 배치하여 균형감을 주고,
    태블릿·모바일에서는 2열 배치 전환으로 일관된 형태를 유지하도록 수정하겠습니다.
    3.3 '오늘의상수도현황' 정수장의 데이터 생략하지 말고 표현바람.✅
    3.4 동복댐에만 표현되어 있는 '댐현황더보기' 기능이 궁금하고, 위치가 애매하고,
    '링크이동'이나 '버튼'이라면 다르게 표현해줬으면 함.
    ✅ '댐현황더보기'는 원안에 없던 임시 요소로 지적 주신대로 삭제하겠습니다.
    3.5 대메뉴가 왼쪽정렬되어 있는 사유가 궁금함.
    --- 디자인 의도 : 시안 1과 동일하게 메뉴 가독성과 로고/검색 영역과의 균형을 위해 좌측 정렬했습니다.
    ✅ 중앙 정렬· 우측 정렬 모두 가능하므로 피드백에 맞춰 조정 가능합니다.
